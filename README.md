# EduLoom (学织) —— 基于大模型多智能体协同的个性化学习资源生成与智能学习系统

> **第十五届“中国软件杯”大学生软件设计大赛 —— A3 赛题参赛项目**
> 
> 本项目致力于利用大语言模型与多智能体（Multi-Agent）技术协同，解决高等教育中学习资源繁杂无序、难以精准匹配个体需求、缺乏智能化与个性化指导的核心痛点。

---

## 🌟 项目简介

**EduLoom (学织)** 是一款专为高等教育打造的个性化学习资源生成与智能辅助系统。我们基于前沿大模型与多智能体协作框架，为每位学生定制专属的个性化资源学习智能体。通过自然语言交互，系统能够精准把握学生的专业背景和知识短板，“因材施教”地自动化生成定制化、多模态的学习资源，并规划科学动态的学习路径，真正实现个性化教育的数字化落地。

### 核心特性
1. **🔒 隐私与数据掌控**：支持完全本地化运行（搭配 Ollama 与 SurrealDB 本地数据库），确保学生的学习数据与个人隐私安全。
2. **🤖 多模型适配**：支持 18+ 主流大模型供应商（包括 OpenAI, Anthropic, Google Gemini, DeepSeek-R1, Ollama 等），可灵活进行成本与效果的优化。
3. **🤝 多智能体协同（Multi-Agent Graph）**：基于 **LangGraph** 框架设计多智能体协同工作流，协同完成多种类型个性化学习资源的生成。
4. **🎙️ 多模态资源生成**：支持将文本材料一键转化为专业的多角色对话播客（Podcast）、讲解文档、思维导图与代码实操案例。

## ✨ 核心业务功能

项目根据“中国软件杯”A3 赛题的基本功能要求进行深度开发，包含以下五大核心系统模块：

### 1. 对话式学习画像自主构建
摒弃传统繁琐的静态表单，学生仅需通过自然语言对话即可完成初始画像的构建。系统依托大模型自动抽取特征，构建包含不少于 **6 个维度** 的动态画像：
*   **知识基础**（先验知识掌握度）
*   **认知风格**（偏好视觉、听觉或代码实操）
*   **易错点偏好**（归纳常见的理解偏差）
*   **学习目标**（考试、科研、工程应用等）
*   **学习进度**（当前章节及吸收效率）
*   **学习兴趣方向**（具体的应用领域偏好）
*   *系统通过实时跟踪后续的学习交互与反馈，实现画像的随学随新。*

### 2. 多智能体协同的资源生成
系统体现了先进的 **Multi-Agent 架构设计**。根据画像中的个性化指标，大模型各司其职，协同生成 5 种以上的个性化学习资源：
*   **专业课程讲解文档**：定制化重构知识难点。
*   **知识点思维导图**：结构化直观展现知识脉络。
*   **多维度练习题库**：生成匹配认知水平的针对性题目及详细解析。
*   **拓展阅读材料**：根据学习兴趣自动搜集并整理业界前沿动态。
*   **多模态音频播客**：一键生成生动有趣的多角色对话式讲解音频。
*   **代码类实操案例**：针对计算机与人工智能相关课程提供可运行的实战代码与详解。

### 3. 个性化路径规划与资源推送
依托多智能体协同工作机制，深度分析学生的学习偏好与进度，为学生定制动态的个性化学习路径，指明学习的步骤和顺序。系统支持多种资源的精准卡片式推送，包含文档、音频播客、题库及实操案例等，并采用**流式输出**与**卡片化渲染**来消除用户的等待焦虑。

### 4. 智能答疑辅导（可选加分项）
学生在学习过程中遇到任何难点，均可随时向系统发起智能提问。辅导智能体结合多模态生成技术与本地向量知识库，为学生提供多样化的即时解惑，例如：文字解答、图解思维导图以及播客讲解片段。

### 5. 学习效果多维评估（可选加分项）
系统实时记录和分析学生的测试练习情况与资源互动频率。大模型多维度评估当前阶段的学习效果，并反馈至画像系统，动态调整后续的学习路径与推送策略。

## 🛠️ 技术架构

EduLoom 采用现代化的全栈 AI 应用技术架构，前后端分离，保障高性能与高响应性。

*   **前端 (Frontend)**：基于 **Next.js** 与 **React** 构建，支持现代 AI 交互规范、流式 Markdown 渲染、卡片化多模态内容展示，响应式设计支持多端设备。
*   **后端 (Backend)**：使用 **FastAPI** 作为 API 服务，利用 **LangChain** 与 **LangGraph** 构建复杂的智能体协同工作流（Workflow & State Machine）。
*   **数据库 (Database)**：采用前沿的 **SurrealDB** 混合多模型数据库（支持 Vector 向量检索与 Graph 图关系存储），能够高效管理多模态资源与学生画像。
*   **核心库名称**：`edu_loom`（项目 Python 包目录）

## 🚀 快速开始

本项目支持本地源码运行，建议使用 `uv` 包管理器以获得最快的依赖安装与运行体验。

### 准备工作

1.  **安装 uv**（Python 高效包管理器）：
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
2.  **安装 SurrealDB**：
    *   macOS (Homebrew):
        ```bash
        brew install surrealdb/tap/surreal
        ```
    *   Windows:
        ```powershell
        iwr -useb https://install.surrealdb.com | iex
        ```
3.  **准备前端环境**：确保本地已安装 **Node.js (>= 18)**。

### 启动步骤

我们提供了一键开发启动脚本，可在本地快速并行拉起所有服务：

```bash
# 赋予脚本可执行权限并运行
chmod +x run-dev.sh
./run-dev.sh
```

`run-dev.sh` 脚本将会：
1. 启动本地 SurrealDB 数据库（默认端口 `8000`）。
2. 在后台拉起 FastAPI 后端 API 服务（默认端口 `5055`）。
3. 启动后台异步任务 Worker。
4. 启动 Next.js 前端开发服务器（默认端口 `3000`）。

启动完成后，请在浏览器中打开：**[http://localhost:3000](http://localhost:3000)**。
